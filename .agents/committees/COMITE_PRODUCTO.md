# COMITÉ DE PRODUCTO

## Justificación de Creación
El proyecto tiene reglas de negocio complejas:
- Multi-tenancy con adaptación de UI
- Constructor de packs (docenas)
- Cross-selling contextual
- Reordenado inteligente
- Múltiples modos de entrega y pago

## Responsabilidades
- Requisitos funcionales y valor de negocio
- Roadmap y priorización
- Experiencia funcional
- Visión de producto

## Roadmap Propuesto

### Fase 1 — Fortalecer Core (P0)
- Validación robusta de formularios
- Notas por producto
- Confirmación visual de envío
- Manejo de errores (WhatsApp no disponible, etc.)

### Fase 2 — Diferenciación (P1)
- Panel de administración mínimo
- Landing page del producto SaaS
- Imágenes locales optimizadas
- PWA + instalable

### Fase 3 — Crecimiento (P2)
- Analytics de uso
- Múltiples categorías configurables
- Gestión de stock básica
- Integración con servicios de delivery

### Fase 4 — Escala (P3)
- Backend ligero para pedidos
- Dashboard para negocios
- Pasarela de pago integrada
- App híbrida (Capacitor/Cordova)
